Research Mode
Improve a measurable research target through failure analysis and targeted fixes. Uses run_command instead of eval_command, enforces mutable/fixed surface constraints, and requires monotonic improvement.

REDIRECT — Vague classifications

R2 Hypothesize Strategist ▶

1-3 hypotheses targeting dominant failures. CEO review includes surface constraint check and ground truth leakage scan.

R2-review: CEO Review — Strategy HARD GATE
PLAN APPROVED — Within surfaces, no leakage, ≤3 hypotheses
REDIRECT — Surface violation or leakage detected
For each hypothesis
R3 Implement Builder ▶

Begin experiment, create issue, Builder implements. MUST only modify mutable_surfaces, MUST NOT modify fixed_surfaces. Violation = automatic revert.

R4 Run Harness Evaluator ▶

Execute run_command on modified code. Single-run or multi-run with aggregation.

R5: Verdict
KEEP — Metric improved + no hygiene regression + precheck passed
REVERT — Metric regressed OR hygiene regressed OR precheck failed
End Termination CEO Decision ▶

Exit when: Target met (metric ≥ target), budget exhausted, or all hypotheses processed. Plateau detection triggers outer loop (expand mutable_surfaces, shift to architectural hypotheses).

Experiment Lifecycle
The inner loop of every experiment: create → execute → finalize. Managed by ExperimentStore.
1 factory begin ▶

Creates experiments/NNN/hypothesis.md, returns EXP_ID, registers project in global registry.

2 Eval Before ▶

save_eval(exp_id, "before", score) → writes experiments/NNN/eval_before.json

3 Builder Implements ▶

Commits code, opens PR. save_diff(exp_id) → writes experiments/NNN/changes.diff

4 Review Pipeline ▶

Full 3-component review: structured checklist → review-until-clean → final headless review.

→ See Review Pipeline

5 Eval After ▶

save_eval(exp_id, "after", score) → writes experiments/NNN/eval_after.json

6 factory finalize ▶

Writes experiments/NNN/verdict.json, appends to results.tsv, updates global registry stats. If verdict=keep and not --force, runs precheck gate which can override to revert.

results.tsv Columns

id | timestamp | hypothesis | change_summary | issue_number | pr_number | score_before | score_after | delta | verdict | cost_usd | notes | research_citations

Review Pipeline
Mandatory for every experiment — no exceptions. Three components ensure code quality before any keep verdict.

Component 1: Structured Checklist

C1 7-Category Code Quality Review CEO Direct ▶

CEO reads PR diff (gh pr diff) and evaluates against:

1. Correctness

Bugs, logic errors

2. Security

Injection, secrets

3. Edge Cases

Empty inputs, timeouts

4. Missing Tests

Untested code paths

5. Style

Naming, dead code

6. Scope

Matches hypothesis

7. Guardrails

File size, scope boundaries, dangerous commands

Verdict

CLEAN → proceed to guard check
ISSUES_FOUND: N → enter review-until-clean loop
ABORT → close PR, finalize as error

Component 2: Review-Until-Clean

Max 3 iterations
C2 Review-Until-Clean Loop Builder (fixes) ▶

1. Check iteration cap (≥3 → stop)
2. Check convergence (issues not decreasing → stop)
3. Route fixes to Builder: "Fix ALL listed issues. No new functionality."
4. Update state, increment iteration
5. Re-run structured review (back to C1)

Component 3: Final Headless Review

C3 Final Headless Review Headless Claude ▶

Runs AFTER all mechanical checks (guard, eval, e2e, precheck). Spawns headless Claude Code to review the complete PR diff for bugs, security, edge cases, error handling, style, dead code.

CLEAN → KEEP approval
ISSUES_FOUND → If iteration ≥3 → KEEP with notes. Otherwise → route fixes, re-run C3 only.

Precheck Gate — 7 Non-Overridable Checks

1
score_direction

Score not regressed AND meets threshold

2
scope

factory guard --check-scope passes

3
fixed_surfaces

factory guard --check-surfaces passes

4
ground_truth_leakage

No content from fixed surfaces leaked

5
anti_pattern

Hypothesis <60% similar to reverted experiments

6
smoke_test

Smoke test command from factory.md passes

7
hard_constraints

User-defined checks from factory.md (each must exit 0)

Agent Roster
8 specialist agents spawned via factory agent <role>. Each gets a fresh context window with its resolved prompt + any evolved playbook auto-injected.
Researcher
Observe: local analysis (factory study) + web research + archive synthesis. Only role that supports parallel spawning via shell backgrounding + --review-tag.
factory agent researcher --task "..." --review-tag <tag>
Strategist
Hypothesize: generate prioritized experiments from observations. In Phase 0 (Ideation): synthesize research + raw idea into buildable spec.
factory agent strategist --task "..."
Builder
Implement: code changes on feature branch, open PR. Enforces scope constraints. Respects mutable/fixed surfaces.
factory agent builder --task "..."
Reviewer
Guard: enforce sacred rules, scope constraints, and code quality on PRs. Substantive review, not rubber-stamp.
factory agent reviewer --task "..."
Evaluator
Measure: run evals before/after changes, report composite + breakdown. Runs both eval_command and run_command (research).
factory agent evaluator --task "..."
Archivist
Record: write learnings to .factory/archive/. MANDATORY at every checkpoint — this is Sacred Rule 7. No exceptions.
factory agent archivist --task "..."
Failure Analyst
Classify failures from research runs by stage and root cause. Produces structured analysis for the Strategist.
factory agent failure_analyst --task "..."
Refiner
Classify and scope refinement requests into tiers. Routes Tier 3 requests to full Improve mode.
factory agent refiner --task "..."

CEO Review Protocol

After every agent completes, the CEO reviews its output:

1. Read the agent's output file (.factory/reviews/<role>-latest.md)
2. Assess against role-specific criteria
3. Write verdict to .factory/reviews/ceo-verdict-<role>.md
4. Act: PROCEED → next step | REDIRECT → re-invoke (max 2) | ABORT → log failure

FEEC Priority Heuristic
Hypotheses are categorized and ranked: Fix → Exploit → Explore → Combine. Backlog items follow the same ordering.
FIX
EXPLOIT
EXPLORE
COMBINE

Fix — Priority 0 (Highest)

Resolve crashes, errors, regressions. Keywords: fix, error, bug, crash, fail, regression, broken, repair.

Exploit — Priority 1

Build on recent successes. Keywords: improve, increase, extend, enhance, build on, optimize, boost.

Explore — Priority 2

Try something new. Default/catch-all when no keywords match.

Combine — Priority 3 (Lowest)

Merge successful patterns. Keywords: combine, merge, integrate, unify, consolidate.

Safety Mechanisms

Stuck Detection

3+ consecutive same-category reverts → MUST pivot to a different FEEC category.

Anti-Pattern Guard

Hypothesis similarity >60% to reverted experiment → blocked by precheck. Uses Jaccard similarity on 3+ char word tokens.

3-Tier History

Last 3: full detail. 4-10: one-line summaries. 11+: aggregate stats only.

Plateau Detection

Last N scored experiments show no improvement → trigger outer loop (expand surfaces, shift to architectural hypotheses).

Sacred Rules
9 inviolable constraints. Checked by factory guard. Violation = automatic revert.
1

Do not delete or overwrite existing tests

Tests may be extended, never removed.

2

Do not modify files outside declared scope

factory.md defines modifiable files.

3

Do not introduce secrets or credentials

No API keys, tokens, or passwords in repo.

4

Do not lower the eval threshold

The bar only goes up.

5

Do not skip the eval step

Every change must be scored before keep.

6

Do not merge PRs

Leave open for human review after KEEP approval.

7

Do not skip archival checkpoints

Archivist must fire at every checkpoint.

8

Do not do another agent's job

CEO orchestrates, delegates, reviews. Never implements.

9

Do not skip the review pipeline

Full 2d-review pipeline for every experiment with a PR.

Keep/Revert Quality Bar

ALL must be true to KEEP:

1. Works correctly — tests pass
2. Observable — changes are logged/traced
3. Evaluated — scores measured before and after
4. Documented — clear commit messages, PR description
5. Maintainable — clean code, no hacks
